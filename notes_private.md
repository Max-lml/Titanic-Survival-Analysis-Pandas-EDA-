# Private notes (not for GitHub)

## Почему в примере был def main()?
- Это не про аналитику, а про структуру программы.
- def main() — собирает весь код в одну функцию.
- if __name__ == "__main__": main() — запускает код только когда файл запускают напрямую.
- Удобно для проектов: чище, понятнее, можно импортировать файл без запуска анализа.

## sort_values / sort_index / ascending
- sort_values() сортирует по значениям (например, доля выживших: сверху самые большие).
  Пример: survival_by_sex.sort_values(ascending=False)
- sort_index() сортирует по индексу (например классы 1,2,3 по порядку).
- Это для читабельности, не обязательно.

## plt.figure()
- Создаёт новое "полотно" (новую фигуру) для отдельного графика.
- Без figure() графики могут налепиться друг на друга.

## Подписи и оформление графиков
- plt.title("...") — заголовок графика
- plt.xlabel("...") — подпись оси X
- plt.ylabel("...") — подпись оси Y
- plt.legend() — показывает легенду, когда на одном графике несколько линий/гистограмм
- plt.ylim(0, 1) — ограничение оси Y (для долей выживаемости, они от 0 до 1)

## dropna()
- Убирает пропущенные значения NaN.
- Нужно перед hist/boxplot, чтобы график не ломался и не искажался.

## alpha
- Прозрачность графика.
- alpha=0.6 означает "чуть прозрачный", удобно когда рисуем 2 распределения на одном графике.

## mean vs median
- mean (среднее) чувствительно к выбросам.
- median (медиана) устойчивее, особенно для Fare (билеты могут быть очень дорогие).
- Поэтому для сравнения цен часто смотрят медиану.

## Что делает merge?
- merge не считает, merge только соединяет таблицы по ключу.
- Счёт делается groupby + agg, затем результат "приклеивается" к users через merge.
